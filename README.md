## Summary
GitHub merupakan **platform version control yang sangat penting dalam pengembangan proyek berbasis tim**. Dengan menggunakan GitHub, tim dapat bekerja secara paralel pada fitur yang berbeda tanpa mengganggu pekerjaan satu sama lain, melacak setiap perubahan kode dengan detail, dan dengan mudah melakukan *tracking bug* jika terjadi kesalahan. Sistem branching memungkinkan pengembang untuk membuat cabang terpisah untuk setiap fitur atau perbaikan bug, sementara pull request memfasilitasi code review yang sistematis sebelum kode digabungkan ke branch utama. Namun, untuk benar-benar memaksimalkan benefit yang didapatkan dari kolaborasi melalui GitHub, **penting untuk menerapkan prinsip berikut**:

- **Memanfaatkan Fitur Branching**
 Memisahkan development, staging, dan production code untuk menjaga stabilitas aplikasi
- **Selalu Pull Request & Code Review** 
Memastikan kualitas kode melalui peer review sebelum merge ke branch utama
- **Resolusi Konflik** 
Pastikan dan diskusikan konflik dengan rekan tim yang terlibat dengan konflik
- **Git Ignore**
Mencegah file-file yang tidak perlu (dependencies, build artifacts, environment files) ter-commit ke repository
- **Commit Message dan Branch Name Convention**
Menggunakan conventional commits (feat, fix, chore, docs, etc.) untuk clarity dan consistency
